▶ Models explained

Logistic Regression

A regularised linear model and the interpretable baseline. All 66 features are z-score normalised before fitting, so the learned coefficients directly quantify each feature's additive contribution to the log-odds of the home team winning. Fast, stable, and fully auditable via coefficient-based SHAP attributions.

  • L2 regularisation (C = 1.0), L-BFGS solver
  • SHAP = coef × z-score, no approximation needed
  • Best for: speed, auditing, explainability

LightGBM

Microsoft's gradient-boosted tree library. Grows an ensemble of shallow trees in sequence — each corrects the errors of the previous ones — capturing non-linear interactions between features (e.g. strong Elo and favourable home/road split together matter more than either alone). Hyperparameters are tuned via a 60-trial Optuna Bayesian search.

  • Optuna-tuned: learning rate, num_leaves, n_estimators, regularisation
  • Tree-based SHAP explanations via TreeExplainer
  • Best for: fast batch inference, non-linear feature interactions

XGBoost

DMLC's gradient-boosted tree library with a different regularisation scheme (separate L1/L2 penalties on leaf weights, min_child_weight). Its split-finding algorithm produces probability estimates that are complementary to LightGBM — the two models tend to disagree most on uncertain games, making them effective ensemble partners. Typically achieves the best single-model Brier score.

  • Optuna-tuned: learning rate, max_depth, n_estimators, regularisation
  • Tree-based SHAP explanations via TreeExplainer
  • Best for: highest single-model accuracy

CatBoost

Yandex's gradient-boosted tree library with ordered boosting and native categorical feature support. Uses symmetric (oblivious) decision trees, which tend to generalise well on tabular data and are resistant to overfitting. Acts as a third complementary tree model in the stacked ensemble.

  • Ordered boosting reduces prediction shift
  • Symmetric trees, L2 leaf regularisation, learning rate decay
  • Best for: robustness, low-variance ensemble partner

Neural Network (MLP)

A multi-layer perceptron classifier with two hidden layers (128, 64 units) and ReLU activations. Captures nonlinear feature interactions that tree models may miss. Features are z-score normalised before training. Provides model diversity for stacking since its error surface is fundamentally different from tree-based learners.

  • Architecture: 128 → 64 → 1, Adam optimizer
  • L2 weight decay for regularisation
  • Best for: ensemble diversity, nonlinear interactions

Stacked Ensemble default

The stacked ensemble never sees raw features. It takes the calibrated probability outputs of all five base models as its inputs and trains a logistic-regression meta-learner to find the optimal blend. Because each base model makes different errors, the meta-learner learns to up-weight whichever model is most reliable in each probability range. Achieves the best overall Brier and calibration.

  • Inputs: Logistic + LightGBM + XGBoost + CatBoost + MLP calibrated probabilities
  • Meta-learner: LogisticRegression (C = 0.5)
  • All base probabilities are out-of-sample relative to the meta-learner
  • Best for: maximum production accuracy

▸ Training techniques
Platt calibration
A sigmoid meta-layer fitted on a held-out calibration set corrects systematic over/under-confidence so that a predicted 65% game actually wins ~65% of the time.
Time-weighted training
Exponential decay (rate = 0.12/season) gives recent seasons more influence. 2024 weight = 1.0 · 2020 weight ≈ 0.61 · 2015 weight ≈ 0.30. Adapts to the 2023 shift ban, pitch clock, and enlarged bases.
Optuna HPO
60-trial Bayesian search over learning rate, tree depth/leaves, n_estimators, subsample, and L1/L2 regularisation — evaluated on a 3-season expanding window to minimise Brier score before the full CV run.
Expanding-window CV
For evaluation season N, the model is trained on all seasons before N. No future data ever leaks into training or calibration. All reported metrics are fully out-of-sample.
